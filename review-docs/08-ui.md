# Review: packages/ui/ (OpenCode v1.2.25 Merge)

## Summary

This merge brings substantial upstream changes to the UI package: a redesigned Card component system with composable sub-components, a new ToolErrorCard component, removal of the session review search infrastructure, addition of fork/revert user actions, a major theme engine overhaul (color scale generation, contrast-aware `on()` function, new AMOLED theme, expanded syntax overrides across all themes), icon additions, i18n updates, and various CSS refinements.

### 24. `src/theme/color.ts`

**Status:** Safe to merge

Theme color generation algorithm updates:

- **`generateScale`:** Light steps for indices 10-11 changed. Dark steps for index 10-11 now use clamped formulas based on base lightness. Chroma multipliers significantly increased across all indices (both light and dark), producing more vibrant/saturated color scales.
- **`generateNeutralScale`:** New `ink?: HexColor` parameter. When provided, generates neutrals by mixing background and ink colors at stepped intervals using a new perceptual blending approach (lift/sink helpers). Without ink, increases neutral chroma cap from 0.02 to 0.04/0.05 (light/dark).

### 27. `src/theme/resolve.ts`

**Status:** Safe to merge — large diff

Major theme resolution overhaul:

- **New `on()` function:** Computes accessible foreground color (white or black) based on WCAG relative luminance contrast ratio against a fill color. Replaces hardcoded `neutralAlpha[10]`/`neutral[11]` for all `text-on-*` and `icon-on-*` tokens.
- **New `lum()` and `hit()` helpers:** WCAG luminance calculation and contrast ratio.
- **New `body` variable:** When ink color is present, computes a body-text color with lightness/chroma shifts for better text contrast.
- **Surface token changes:** `surface-float-base` and `surface-float-base-hover` now use `neutral[1]`/`neutral[2]` for ink themes in dark mode. No-ink surface alpha values uniformly increased (higher contrast surfaces).
- **Border tokens:** Shifted up one index across the board (e.g., `interactive[6]` → `interactive[7]`), producing stronger borders.
- **Syntax highlighting:** `syntax-keyword` now uses `content(colors.accent, accent)` instead of `"var(--text-weak)"` for compact themes. `syntax-primitive` now uses `content(colors.primary, primary)`. `syntax-property` uses `content(colors.info, info)`. `syntax-constant` uses `content(colors.accent, accent)`. Syntax status colors shifted from index 9 to 10.
- **Compact ink backgrounds:** Shifted to darker indices (`neutral[0]` instead of `neutral[2]`).
- **`generateNeutralAlphaScale`:** Completely rewritten — now uses simple `blend()` between scale endpoints instead of per-index oklch mixing. Alpha steps significantly increased.
- **`content()` helper:** Reworked — dark mode now checks if `base.l > 0.84` to decide between shifted seed or scale[10]; both modes apply additional lightness and chroma shifts.
- **`brand`/`interactive`/`success`/`warning`/`critical`/`info` surface tokens:** Extracted into short local variables (`brandb`, `interb`, `succb`, etc.) for readability and reuse with `on()`.

## Cross-Cutting Concerns

### Kilo-specific changes

- `card.css` has a properly annotated `kilocode_change` marker on the `::before` accent line (set to transparent instead of upstream's `var(--card-accent)`).
- No other Kilo-specific modifications detected in this package.

### Breaking changes

- `FileSearchHandle` API reduced to just `{ focus }` — any Kilo code passing `setQuery`, `clear`, `reveal`, `expand`, or `refresh` will break.
- `FileSearchControl` reduced — `shortcuts`, `showBar`, `disableVirtualization` removed.
- `MessageProps` no longer has `queued` prop; replaced by `actions` prop.
- `SessionTurn` no longer has `queued` prop; has `actions` prop.
- `session-review-search.ts` and its test deleted — any imports will break.

### Potential issues

1. **Hardcoded string in ToolErrorCard:** `"Copy error"` on line 117 of `tool-error-card.tsx` should ideally be an i18n key.
2. **`nl.ts` and `uk.ts`:** New `forkMessage`/`revertMessage` keys are English-only and not translated.

---

## Verdict

**Approve.** All changes are upstream OpenCode updates that integrate cleanly. The one Kilo-specific override in `card.css` is properly annotated. The theme engine overhaul is substantial but well-structured, and the removal of session review search is clean with no orphaned references. Breaking API changes to `FileSearchHandle`/`FileSearchControl` and message props need to be verified against Kilo consumers (VS Code extension, Agent Manager).
